# Release Notes - v0.2.0

## Overview

This release introduces improved plugin identification and enhanced configuration flexibility for route file extensions.

## üöÄ New Features

### Custom Entrypoint Extensions

- **New `entrypointExtensions` Option** - Added a new configuration property that allows you to specify custom file extensions for route entrypoints
- **Default Extensions** - Defaults to `[".tsx", ".jsx"]` for backward compatibility
- **Greater Flexibility** - Enables support for different file naming conventions in your routing directory

```typescript
applyReactPluginToHTML({
  style: "nextjs",
  route: "src/pages",
  entrypointExtensions: [".tsx", ".jsx", ".ts"], // Custom extensions
});
```

## üîß Improvements

### Plugin Identification

- **Package Name Integration** - The plugin name is now automatically set to the package name (`frame-master-plugin-apply-react`) for better identification in logs and debugging

## üìù Summary

Version 0.2.0 focuses on improving plugin modularity and identification. The new `entrypointExtensions` option provides greater control over which file types are recognized as route entrypoints, while the plugin naming improvement enhances debugging and tooling integration.
